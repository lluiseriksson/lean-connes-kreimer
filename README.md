# lean-connes-kreimer

Lean 4 + Mathlib scaffold for the Connes-Kreimer Hopf algebra of rooted
trees.  This is a satellite repository for the broader THE-ERIKSSON-PROGRAMME,
but it currently has no direct import path into that repository.

## Scope disclaimer

This repository is about the combinatorial Hopf algebra of Connes and Kreimer.
It does not prove, claim, or consume any Yang-Mills mass-gap statement.  A
future consumer may be the counterterm bookkeeping layer if `hRpoly` develops
into a genuinely multiscale renormalization component.  Until such a consumer
exists, there is no claimed connection to Yang-Mills renormalization.

## Current state

* Toolchain: `leanprover/lean4:v4.29.0-rc6`.
* Mathlib pin: `07642720480157414db592fa85b626dafb71355b`, copied from
  `github.com/lluiseriksson/THE-ERIKSSON-PROGRAMME`.
* `ConnesKreimer.Basic` gives the no-sorry M0 core:
  * `Forest Tree := Finsupp Tree Nat`;
  * `CKAlgebra R Tree := MvPolynomial Tree R`;
  * forest vertex counts from an explicit `Tree -> Nat`;
  * a predicate for homogeneous vertex degree.
* `ConnesKreimer.Interfaces` is the stable import surface for future consumers.

## Honesty policy

Main must stay free of `sorry` and project axioms.  Frontier statements with
unfinished proofs belong on branches named `frontier/*`, and every unproved
mathematical input must appear as an explicit theorem argument or structure
field, never as an `axiom`.

See `HYPOTHESIS_FRONTIER.md` for the current frontier and `MATHLIB_AUDIT.md`
for the pre-build audit.

## Milestones

* M0: free commutative algebra generated by rooted trees, with grading by
  vertex number.
* M1: coproduct by admissible cuts, counit, and coassociativity.
* M2: connected graded bialgebra gives the recursive antipode and the full
  Hopf algebra.
* M3 stretch: Birkhoff decomposition and a toy BPHZ demo for combinatorial
  Feynman rules.
